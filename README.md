# Prompt Engineering Library for PMs

A practical, PM-focused library of prompts designed to accelerate project delivery using AI tools.  

This collection demonstrates how project managers can use prompt engineering to improve clarity, speed, and alignment across requirements, user stories, testing, process mapping, and portfolio reporting.

---

## 📁 What’s Inside

- **Requirements Prompts** — Prompts for eliciting, clarifying, prioritizing, and validating business and technical requirements.
- **User Story Prompts** — Prompts for generating user stories, improving quality, and creating acceptance criteria.
- **Test Case Prompts** — Prompts for generating test cases, regression scenarios, UAT flows, and risk-based testing.
- **Process Mapping Prompts** — Prompts for documenting current-state workflows, designing future-state processes, and identifying automation opportunities.
- **Portfolio Reporting Prompts** — Prompts for creating executive-ready status summaries, KPI narratives, risk reports, and portfolio roll-ups.

---

## 🎯 Why This Matters

Prompt engineering is now an essential skill for modern project managers. It enables faster analysis, clearer documentation, and stronger alignment across teams. 

This library demonstrates how AI can streamline core PM workflows—from requirements to reporting—so projects move with greater speed, consistency, and confidence.

### Key Points
- **Accelerates requirements** — quickly clarifies needs and reduces ambiguity early in the project.
- **Improves documentation quality** — produces clearer user stories, test cases, and process maps.
- **Strengthens alignment** — ensures teams share a consistent understanding of goals and scope.
- **Speeds decision-making** — surfaces risks, options, and insights faster.
- **Enhances reporting** — generates polished, executive-ready summaries with minimal manual effort.
- **Reduces cycle time** — shortens the path from idea → requirement → delivery.
