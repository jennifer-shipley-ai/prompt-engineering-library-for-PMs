# User Story Prompts for PMs

Prompts designed to generate, refine, and validate user stories with precision and consistency.

---

## 1. Generate User Stories from Requirements
**Prompt:**  
“Convert these requirements into user stories using the format: As a <role>, I want <action>, so that <value>.”

---

## 2. Improve User Story Quality
**Prompt:**  
“Evaluate this user story for clarity, completeness, and testability. Suggest improvements and rewrite it in best-practice format.”

---

## 3. Acceptance Criteria Generation
**Prompt:**  
“Generate acceptance criteria for this user story using Gherkin format (Given/When/Then). Include edge cases.”

---

## 4. Identify Missing User Stories
**Prompt:**  
“Analyze this workflow and identify any missing user stories needed to support the full end-to-end experience.”

---

## 5. Role-Based Story Expansion
**Prompt:**  
“Expand this user story by generating variations for different user roles (admin, end user, manager, auditor).”

---

These prompts help PMs create consistent, high-quality user stories that accelerate delivery.
